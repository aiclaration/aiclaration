## Neue Findings (priorisiert, mit Beweis)

### 🔴 1. `/api/subscribe` ohne Missbrauchsschutz — E-Mail-Bombing-Vektor
**Befund:** [src/app/api/subscribe/route.ts](../src/app/api/subscribe/route.ts) hat — anders als `/api/validate` — **kein Rate-Limit, kein Honeypot, kein Captcha**. Der Endpoint löst bei jeder eingegebenen Adresse eine Brevo-DOI-Mail aus.
**Risiko:** Angreifer kann beliebige Adressen über die eigene Domain mit Bestätigungsmails bombardieren → Brevo-Kontingent + Sender-Reputation verbrannt, genau in der Phase, deren Ziel Deliverability ist.
**Fix (offen):** Rate-Limit analog `/api/validate` (in-memory, per-IP) + verstecktes Honeypot-Feld im Formular.

### 🟡 2. Stiller Erfolg bei fehlender Konfiguration
**Befund:** [route.ts:29-33](../src/app/api/subscribe/route.ts#L29-L33) — fehlen die Brevo-Env-Vars, Rückgabe `{message:'ok'}`/200. Nutzer sieht „Fast geschafft!", obwohl nichts gespeichert wurde.
**Risiko:** Fehlkonfiguriertes Deployment verliert jede Anmeldung lautlos — KPI „100 E-Mails" ohne Signal gefährdet.
**Fix (offen):** In Produktion (`NODE_ENV === 'production'`) laut fehlschlagen (500 + Log), nur in Dev schlucken.

### 🔴 3. Null automatisierte Tests
**Befund:** Keine einzige `*.test.ts`/`*.spec.ts` im Projekt (nur in node_modules). Pflichtenheft S9.2/S9.3 **verlangt** 10 Schema- + 8 SSRF-Szenarien. Sicherheitskritischer Code ([ssrf.ts](../src/lib/ssrf.ts), [schema.ts](../src/lib/schema.ts)) ohne Regressionsnetz; „manuell einmal geprüft" (Session 04) überlebt die nächste Änderung nicht.
**Fix (offen):** Vitest-Suites für SSRF + Schema; danach E2E (Playwright) für Generator/Validator/E-Mail-Form.

### 🟡 4. Spec-Defekt `labeling_method: 'ai Generated'` weiterhin live
**Befund:** [schema.ts:36](../src/lib/schema.ts#L36) enthält weiterhin das malformte Enum (Leerzeichen + Großbuchstabe). Seit Session 03/04 geflaggt, korrekt nicht eigenmächtig geändert.
**Fix (offen):** Founder-Freigabe → Pflichtenheft auf `'ai_generated'` korrigieren, dann Schema angleichen.

### 🟡 5. Ungeprüfte Bereiche (ehrliche „unknown unknowns")
Kein Dev-Server/Browser-Test diese Session → Lighthouse, a11y-Audit und echte E2E sind **ungeprüft**, nicht „grün". Keine Analytics → 100-E-Mail-Funnel ist blind (bewusste Privacy-Entscheidung, hat aber einen Preis).
